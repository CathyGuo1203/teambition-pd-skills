# Teambition 产品核心概念参考

这份文档旨在为 AI 技能提供关于 Teambition 产品的核心上下文，以确保生成的 PRD 在功能、术语和逻辑上与产品保持一致。

---

## 1. 核心对象与层级

Teambition 的核心数据结构围绕以下层级展开：

1.  **企业 (Organization)**: 顶层单位，包含所有项目、成员和资源。
2.  **项目 (Project)**: 工作的基本单元，用于组织和管理一系列相关的任务。
3.  **任务分组 (Task Group / List)**: 在项目中对任务进行分类的容器，常用于表示工作的不同阶段或类别（如"待处理"、"进行中"、"已完成"）。
4.  **任务 (Task)**: 最基本的工作项，可以分配给成员并设置各种属性。
5.  **子任务 (Sub-task)**: 任务的下一级，用于将一个复杂的任务分解为更小的、可执行的部分。

---

## 2. 任务 (Task) 的核心字段

每个任务都包含一系列标准字段和自定义字段。

### 2.1 标准字段

| 字段名称 | 英文标识 (推测) | 数据类型 | 说明 |
| --- | --- | --- | --- |
| **标题** | title | String | 任务的名称，必填。 |
| **执行者** | executor | Member | 任务的负责人，唯一。 |
| **参与者** | participants | Array<Member> | 任务的协作者，可多个。 |
| **截止日期** | dueDate | Datetime | 任务的完成时限。 |
| **优先级** | priority | Enum | 任务的紧急程度（如：普通、紧急、非常紧急）。 |
| **任务状态** | status | Enum | 任务在工作流中的当前阶段，与任务分组/看板列关联。 |
| **标签** | tags | Array<String> | 为任务添加分类或关键词。 |
| **备注** | note | String | 对任务的详细描述或补充信息。 |
| **关联内容** | linkedContent | Array<Link> | 关联到其他对象，如分享、文件、日程等。 |
| **父任务** | parentTask | Task | 指向其所属的父任务。 |

### 2.2 自定义字段

Teambition 支持丰富的自定义字段类型，以适应不同团队的工作流需求。

| 字段类型 | 英文标识 (推测) | 说明 |
| --- | --- | --- |
| **文本** | text | 单行或多行文本输入。 |
| **数字** | number | 用于输入数值，支持设置单位和精度。 |
| **日期** | date | 用于选择日期或日期范围。 |
| **单选菜单** | select | 从预设选项中选择一个。 |
| **多选菜单** | multiSelect | 从预设选项中选择多个。 |
| **成员** | member | 从项目成员中选择一个或多个。 |
| **关联任务** | task | 关联到项目中的其他任务。 |
| **附件** | attachment | 上传文件或图片。 |

**注意**: 自定义字段支持设置"提示文案"和"默认值"，这在设计新功能时需要考虑。

## 3. 项目 (Project) 的核心特性

### 3.1 项目视图

项目中的任务可以以多种视图呈现，以适应不同的管理和浏览需求。

- **列表 (List)**: 以清单形式展示任务，清晰直观。
- **看板 (Kanban)**: 以卡片形式在不同列之间拖动，适合流程管理。
- **表格 (Table)**: 类似电子表格，可以批量编辑任务字段。
- **甘特图 (Gantt)**: 可视化任务排期、依赖关系和项目进度。
- **日历 (Calendar)**: 在日历上展示有截止日期的任务。

### 3.2 项目角色与权限

| 角色名称 | 说明 |
| --- | --- |
| **拥有者 (Owner)** | 拥有项目的最高权限，可以转让项目、删除项目。 |
| **管理员 (Admin)** | 拥有除转让/删除项目外的所有管理权限，如管理成员、编辑项目信息等。 |
| **成员 (Member)** | 拥有任务的创建、编辑、评论等基本操作权限，但不能管理项目设置。 |

**注意**: 权限控制粒度主要在**项目级别**，部分操作可在企业级别进行统一配置。

## 4. 自动化 (Automation)

自动化是 Teambition 的核心高级功能，基于"触发器-条件-动作"模型。

### 4.1 触发器 (Triggers)

当满足特定条件时，自动触发一个规则。

- **任务被创建时**
- **任务被更新时** (例如：执行者变更、截止日期变更、状态变更)
- **截止日期到期时**
- **评论被创建时**
- **成员被添加/移除时**

### 4.2 条件 (Conditions)

用于筛选触发规则的任务范围。

- **任务属性**: 任务类型、任务层级、任务分组、任务列表等。
- **任务字段**: 执行者、截止时间、优先级、标签、自定义字段等。

### 4.3 动作 (Actions)

当规则被触发时，系统自动执行的操作。

- **更新任务**: 更改执行者、设置截止日期、添加标签、修改自定义字段值。
- **移动任务**: 将任务移动到其他任务分组或任务列表。
- **创建子任务**
- **发送通知**: @ 成员或发送自定义提醒。
- **跨任务执行**: 更改父任务、子任务或关联任务的字段（旗舰版功能）。

Teambition 主要服务于以下行业的中小企业团队：

| 行业 | 典型团队类型 |

| 金融软件 | 研发团队、项目交付团队 |

| 汽车制造 | 研发团队、市场团队 |

## 6. 常用术语对照

在撰写 PRD 时，请使用以下 Teambition 官方术语，避免使用同义词或竞品术语。

| Teambition 术语 | 避免使用 | 说明 |
| --- | --- | --- |
| 任务 (Task) | 工单、卡片、Issue | Teambition 的基本工作单元 |
| 执行者 | 负责人、Assignee | 任务当前阶段的唯一负责人 |
| 参与者 | 协作者、关注者 | 参与任务但不是主要负责人 |
| 任务分组 | 列表、泳道、阶段 | 项目中对任务进行分类的容器 |
| 任务状态 | 进度、阶段 | 任务在工作流中的当前位置 |
| 自动化 | 工作流自动化、触发器 | Teambition 的自动化功能模块名称 |
| 项目 | 工作区、空间 | Teambition 中组织任务的基本单元 |
